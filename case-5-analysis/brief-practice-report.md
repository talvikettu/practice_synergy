# Краткий отчет по реализованным кейсам

## Кейс-задача № 1

В первом задании необходимо было создать программу для стилистического преобразования чисел. Данная программа была реализована на языке C++ с использованием стандартных библиотек языка. В ходе выполнения задания были реализованы следующие функции:

1. Запрос у пользователя дня рождения, месяца и года рождения с обработкой ошибок и некорректного ввода.
2. Определение дня недели, на который приходится введенная дата рождения.
3. Определение, является ли год рождения пользователя високосным.
4. Определение текущего возраста пользователя на основании введенной даты рождения.
5. Вывод даты рождения в консоль в формате `дд мм гггг`, где цифры стилизованы с помощью символов `*` по аналогии с электронным табло.

В результате была создана консольная программа, которая корректно обрабатывает пользовательский ввод, выполняет вычисления по дате и выводит результат в наглядной текстовой форме.

Ссылка на репозиторий с исходным кодом, README-файлом, а также со скомпилированным приложением: `practice_synergy/case-1-cpp at master · talvikettu/practice_synergy`.

## Кейс-задача № 2

Во втором задании необходимо было создать простейший счетчик с использованием стилизации интерфейса. Данное приложение было реализовано с помощью HTML, CSS и ванильного JavaScript. В ходе выполнения задания были реализованы следующие элементы и функции:

1. Создан пользовательский интерфейс, состоящий из двух кнопок: «плюс» и «минус», а также окна для отображения результата.
2. Добавлена стилизация интерфейса: градиентный фон страницы, выравнивание элементов по центру окна браузера, скругление кнопок и использование градиента в оформлении кнопок.
3. Реализован функционал увеличения и уменьшения значения счетчика на единицу с помощью обработчиков событий.
4. Добавлено изменение цвета окна результата в зависимости от значения счетчика: желтый цвет для положительных значений, зеленый для отрицательных и красный для нуля.
5. Реализовано ограничение диапазона значений от `-10` до `10`, при достижении которых соответствующая кнопка становится неактивной.
6. Добавлено отдельное текстовое окно, в котором выводится сообщение о достижении экстремального значения.

В результате было создано интерактивное веб-приложение с понятным интерфейсом и динамическим поведением, соответствующее поставленным требованиям.

Ссылка на репозиторий с исходным кодом и README-файлом: `practice_synergy/case-2-vanilla-js at master · talvikettu/practice_synergy`.

## Кейс-задача № 3

В третьем задании необходимо было создать веб-приложение на фреймворке Django, позволяющее пользователю вводить свое имя и получать персонализированное приветствие. В ходе выполнения задания были реализованы следующие компоненты:

1. Создана главная страница с формой ввода имени и кнопкой `Submit`.
2. Разработана модель Django для хранения имен пользователей в базе данных SQLite.
3. Реализована серверная логика, принимающая имя из формы, сохраняющая его в базе данных и отображающая персонализированное приветствие на главной странице.
4. Добавлена обработка ошибок, включая проверку на отправку пустого поля имени.
5. Выполнена стилизация страницы с использованием CSS для улучшения визуального оформления формы, кнопок и блоков вывода данных.
6. Обеспечено соответствие базовым требованиям безопасности, включая защиту формы от CSRF-атак средствами Django.
7. Дополнительно реализован вывод списка последних сохраненных пользователей, что делает приложение более наглядным и удобным при тестировании.

В результате было создано работоспособное веб-приложение на Django с подключенной базой данных, валидацией формы, защитой от CSRF и сохранением введенных данных.

Ссылка на репозиторий с исходным кодом, README-файлом и инструкцией по ручному тестированию: `practice_synergy/case-3-django at master · talvikettu/practice_synergy`.

## Кейс-задача № 4

В четвертом задании необходимо было разработать интерактивный калькулятор. Для реализации данного кейса было выбрано настольное приложение на WPF с использованием языка C#. В ходе выполнения задания были реализованы следующие элементы и функции:

1. Создан графический интерфейс, включающий два поля ввода, четыре кнопки математических операций и окно вывода результата.
2. Выполнена стилизация интерфейса с использованием элементов WPF и индивидуального оформления кнопок и области результата.
3. Реализованы четыре функции математических операций: сумма, разность, произведение и деление.
4. Добавлены обработчики событий, позволяющие выполнять выбранную операцию после ввода двух чисел.
5. Реализована проверка корректности ввода с выводом сообщения об ошибке, если пользователь вводит нечисловые значения.
6. Дополнительно добавана обработка деления на ноль для предотвращения некорректной работы программы.

В результате было создано удобное настольное приложение-калькулятор с графическим интерфейсом, обработкой ошибок и поддержкой базовых математических операций.

Ссылка на репозиторий с исходным кодом, README-файлом и собранным приложением: `practice_synergy/case-4-wpf at master · talvikettu/practice_synergy`.
